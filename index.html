Overview

piDisplay is a 16x2 character LCD based minecraft server notification system. You will need a Minecraft server running this plugin and build the hardware decsribed in this guide.

How to use

If you are not interested in playing with the code then just download the latest .jar file from the releases link. After downloading, copy it to your minecraft server plugins folder. Guidance on registering Twitter applications here.

Once complete you will need to note your Consumer Key (API Key), Consumer Secret (API Secret), Access Token and Access Token Secret. These should appear as long strings and you need to enter them into the plugin settings in order for the automation to work.

Upon first run the plugin will write a config.yml to its subdirectory in your plugins folder. The default content for this file is as below
TWITTER_CONFIGURED: false
API_KEY: XXXX
API_SECRET: YYYY
token: ZZZ
secret: ABABAB

    Edit the file directly to use the keys saved from Twitter and change TWITTER_CONFIGURED to true to enable tweeting. Restart the server or reload the plugin using the console command.

Detailed configuration

The plugin has 3 commands that can control events are tweeted.

    listNotification
    Lists the current settings.
    setNotification [Notification name] [Status]
    Sets any individual setting eg
    loggingInOut
    blockPlacing
    dying
    taming
    fishing
    kicking
    teleporting
    enteringVehicle
    duplicate
    Setting any of the above to true or false switches the relevant notification to on or off. Default setting is true for all of them except blockPlacing and duplicates which are both false by default. Twitter has a limit of 180 tweets per 15 minute window so having blockPlacing switched on will easily exceed the limit when building. If this happens then the plugin will stop tweeting until the window is reopened.
    updateMineTwitConfig
    Updates from config file

It should be noted that restarting the server after manually editing config.yml will have broadly the same effect except that in the case of an error it will revert to the defaults not the last known good value. For this reason it is recommended to either use the console commands to change parameters or use updateMineTwitConfig if manually editing.
